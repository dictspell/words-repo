Engineering Depth

Battle-Tested in Production

Real bugs we've found and fixed in CDC pipelines — the kind of issues that silently corrupt your data.

CRITICAL

Silent Sync Hang

SQL cache keyed only by table name. When Debezium omitted NULL columns, the cached SQL had wrong placeholders — silently swallowed errors. No logs. No warnings. Just a hung job.

DATA INTEGRITY

Lost Decimal Scale

Debezium encodes NUMBER(10,2) as base64 without scale metadata. A value of 5000.00 was decoded as 500000 — violating CHECK constraints and silently corrupting financial data.

DATA INTEGRITY

Case-Sensitive Column Sort

SQL builders sorted UPPERCASED names, but parameter binding sorted original-case JSON keys. Java's case-sensitive sort caused wrong values in wrong columns — silent data corruption.

RELIABILITY

Cross-Table Batch Contamination

All tables shared one JDBC connection and transaction. One bad row in CUSTOMERS rolled back valid ADDRESSES rows. Multiple tables appeared hung together.

MEMORY LEAK

Infinite FK Retry Loop

FK retry queue created new entries with retryCount=0 on each failure. The original count was discarded. Max retries were never reached — eventual OutOfMemoryError.
